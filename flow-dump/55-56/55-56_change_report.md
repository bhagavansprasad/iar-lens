## 2. What This Integration Does

This integration is a scheduled process designed to collect inventory data from various external systems, transform it, and store snapshots in a database for reconciliation. Its primary goal is to calculate inventory differences and synchronize the updated, reconciled inventory information with Oracle Inventory. This ensures Oracle always reflects the latest and most accurate inventory status from multiple supply chain partners.

**Adapter Changes:** **Added:** `ArchiveDhlFile`, `DownloadDHLFile`, `InsertDhlIntoSnapshotTbl`, `ListDHlFiles`, `ListFiles` | **Removed:** `CallOracleInventorySync`

## 3. What Changed

**Summary of Change:**
This update significantly expands the integration's capabilities by incorporating a new data source: DHL inventory files. Previously, the system reconciled inventory using general ERP and Enterprise Inventory data. With this change, it now includes DHL inventory, creating a dedicated snapshot for it, which provides a more holistic and accurate view of the overall inventory. The removal of specific legacy error handling for Oracle file generation suggests a simplification or improvement in how Oracle synchronization is managed. The overall impact is a richer, more complete, and more robust inventory reconciliation process that now includes an additional key logistics partner.

**Before (v03.00.0001):**
The integration started on a schedule, initialized variables, and then processed a primary inventory file, likely from an ERP system, capturing this data as an ERP snapshot while handling errors. Next, it looped through and processed multiple Enterprise Inventory (EI) files, saving their data as an EI snapshot. After processing all available inventory sources, it performed final transformations and assignments, then identified differences between the snapshots, and finally synchronized the reconciled inventory with Oracle Inventory. A separate section handled specific error conditions related to Oracle file generation or when no relevant files were found.

**After (v03.00.0011):**
The updated integration also starts on a schedule, initializes variables, and now processes primary inventory files from potentially multiple directories or sources, creating an ERP snapshot. It then proceeds to process Enterprise Inventory (EI) files, similar to before, creating an EI snapshot. Crucially, a new section has been added to ingest and process specific DHL inventory files, storing them in a dedicated DHL snapshot. After consolidating data from all these sources (ERP, EI, and DHL), it reconciles the combined inventory, calculates differences, and then synchronizes the comprehensive, reconciled inventory with Oracle. The legacy error handling for Oracle file generation has been removed, indicating a more streamlined approach.
